## 一、 视觉规范 (Design System)

### 1. 调色盘 (Color Palette)
使用 Lipgloss 的十六进制颜色定义，确保在高对比度下的美感。

| 角色 | 颜色代码 | 视觉用途 |
| :--- | :--- | :--- |
| **背景色 (Primary BG)** | `#1A1B26` | 全局底色，深邃的午夜蓝 |
| **强调色 (Accent)** | `#7AA2F7` | 焦点、高亮、主要按钮颜色 |
| **行为字段 (Action)** | `#BB9AF7` | `source_app_type` 及逻辑层级高亮 |
| **字典已映射 (Match)** | `#9ECE6A` | 成功映射的中文、成功状态状态 |
| **字典未映射 (Missing)** | `#F7768E` | 告警、未关联字典的错误高亮 |
| **辅助文本 (Muted)** | `#565F89` | 快捷键提示、次要说明、边框颜色 |

### 2. 边框与间距 (Layout)
*   **全局边框**：使用 `Rounded` (圆角) 风格边框，厚度为 1px。
*   **内边距 (Padding)**：所有容器保持 `Left/Right: 2`, `Top/Bottom: 1` 的呼吸感。
*   **阴影感**：通过 Lipgloss 的 `Shadow` 或背景色偏移模拟层级。

### 3. 图标库 (Typography & Icons)
推荐用户安装 `JetBrainsMono Nerd Font`。
*   `󰚭` (ES) | `󰆼` (DB) | `󰍉` (Search) | `󰄬` (Success) | `󰅖` (Error) | `󰏫` (Manual Mapping)

## 二、 页面布局与功能交互

### 1. 闪屏预检 (Splash & Initializer)
*   **布局**：水平垂直居中。
*   **UI表现**：
    *   使用渐变色的 ASCII 艺术 Logo "ES-SPECTRE"。
    *   下方是一个动态进度条 (`Progress Bar`)。
    *   右侧滚动显示自检日志：`[OK] Loading ES Config...`, `[OK] Detecting Dameng Driver...`。
*   **交互**：检测失败时，进度条变红，并弹出错误气泡，提示按 `R` 重试或 `E` 修改配置。

### 2. 连接工作台 (Connection Center)
*   **布局**：左侧 30% 为“基础设施列表”，右侧 70% 为“详细参数看板”。
*   **元素字段**：
    *   **ES 状态卡片**：显示集群名称、可用索引。
    *   **DB 状态卡片**：显示数据库类型（图标识别）、连接延迟。
    *   **驱动挂载点**：列出 `./drivers` 下的 `.dll/.so` 文件，绿色表示已加载。
*   **交互**：方向键切换，选中项边框呈现淡蓝色呼吸灯特效。

### 3. 数据探索器 (Field Explorer)
*   **布局**：双栏设计。
    *   **左侧 (Field List)**：带滚动条的长列表。每一行格式：`[ ] field_name (type)`。
    *   **右侧 (Preview)**：选中某个字段时，展示该字段在 ES 中的前 3 条样本数据，帮助用户判断是否是目标字段。
*   **组件开发细节**：
    *   **顶部搜索框**：固定在顶端，输入即时过滤左侧列表（Fuzzy Search）。
    *   **操作栏 (Bottom Bar)**：`Space` 勾选，`Enter` 确认，`A` 全选。
    *   **勾选反馈**：勾选后的字段背景变为深蓝色。

### 4. 字典映射实验室 (Mapping Lab)
这是交互最密集的模块。
*   **布局**：全屏网格。表格列：`字段名` -> `映射状态` -> `关联字典编码` -> `样例文本`。
*   **状态设计**：
    *   **Auto-Match (绿色)**：匹配成功的，显示对应的 `item_text` 样例。
    *   **Unmapped (红色)**：显示 `[!] 缺失映射`。
*   **弹窗交互 (Modal)**：
    *   选中红色项按 `Enter` 弹出半透明悬浮层。
    *   **输入模式**：实时匹配数据库中的 `dict_code`。
    *   **列表滚动**：展示数据库字典库的搜索结果。
    *   **确认反馈**：映射成功后，行背景由红转绿。

### 5. 主分析看板 (Analysis Dashboard) - "Ghost Grid"
*   **核心组件**：**阶梯式多维数据表 (Stepped Multi-Dim Grid)**。
*   **视觉层级**：
    *   **一级行 (Behavior)**：加粗、带图标。例如：`󰚭 主机硬件变更 [842次, 42.5%]`。
    *   **二级行 (Sub-field)**：向右缩进 4 格，边框垂直连线。例如：`┣━ 屏幕 [300次]`。
    *   **三级行 (Detail)**：再缩进 4 格。
*   **图形增强**：每一行末尾显示一个字符画 Mini-Bar (`██████░░░░ 60%`)，颜色随百分比从青到蓝过渡。
*   **快捷操作**：
    *   `↑/↓`：平滑滚动。
    *   `Ctrl+E`：触发导出（出现微小的 Loading 动画）。
    *   `F`：切换全屏/常规视图。

## 三、 动态交互与微动画 (Micro-Animations)

1.  **Spinner (加载中)**：查询 ES 时，在屏幕右下角显示一个精致的 `Dots` 型动画。
2.  **Slide In (切页自愈)**：页面切换时，使用 Bubble Tea 的窗口大小调整机制，实现内容从左向右滑动的视觉错觉。
3.  **Toasts (吐司提示)**：导出成功后，屏幕顶端滑下一个绿色小框：`󰄬 Exported to /export/report_2023.xlsx`，3秒后自动淡出。
4.  **Cursor (光标点缀)**：被选中的行前方不只是变色，可以增加一个渐变色的指示箭头 `»`。

## 四、 开发实现建议 (Developer Tips)

1.  **样式复用**：在 `internal/ui/styles.go` 中统一定义所有 Lipgloss 变量，严禁在渲染函数中写死颜色。
2.  **状态机设计**：
    *   `StateInitial`: 闪屏。
    *   `StateConfig`: 连接。
    *   `StateExplore`: 选字段。
    *   `StateMap`: 字典处理。
    *   `StateResult`: 最终看板。
3.  **防抖处理**：在 Mapping 搜索界面，对数据库输入查询做 300ms 的防抖，避免瞬间大量 SQL 查询压力。
4.  **虚拟化渲染**：如果 ES Mapping 超过 200 个字段，请确保使用 `list` 组件的虚拟化功能，只渲染当前视口，保证 60FPS。
